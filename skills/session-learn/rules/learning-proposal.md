# Learning Proposal Schema (Scope-Attached Rule File)

Every proposal generated by `/session-learn` MUST use this exact shape.

## Required structure per proposal

```markdown
### Proposal: <one-line summary>

**Category:** [HARD-RULE | AGENT-IMPROVEMENT | NEW-SKILL | PROJECT-FACT | ANTI-PATTERN | GAP-FLAG]

**Target file:** `<full path>` (or "new file: <path>" for new-skill proposals)

**Evidence from this session:**
> "<quote one specific moment from the session — user correction, model failure, repeated pattern, etc.>"

[Optional additional quotes if pattern repeated]

**Pattern detected:** [One sentence — what this evidence shows]

**Current text to change:** (omit if target is a new file or new CLAUDE.md section)
```
[Exact existing snippet]
```

**Proposed new text:**
```
[Exact replacement or new content]
```

**Reasoning:** [2-3 sentences. Why this addresses the pattern. What it will and will not change about future sessions.]

**Risk:** [LOW | MEDIUM | HIGH — one sentence on what could go wrong]

**Reversibility:** [Easy via git? Yes/No]
```

## Rules for the analyser

1. **One file per proposal.** If a learning affects multiple files, generate multiple linked proposals.
2. **Evidence must be quoted, not summarised.** The session moment goes in quote marks.
3. **No speculative learnings.** If you can't quote a specific moment, you don't have evidence.
4. **CLAUDE.md additions must match the user's voice** — British English, direct, no em dashes, no AI-corporate phrasing. If the user writes in bullets, propose bullets. If in tables, propose tables.
5. **Agent improvements must respect the agent's defined role** — sharpen, do not redirect. Frontmatter is sacrosanct.
6. **New skill proposals must justify why** — only propose if the pattern repeated three or more times in the session, or the user explicitly asked for it.
7. **GAP-FLAG proposals are flag-only.** They are noted in the learnings doc but apply no file change — they exist to populate a backlog of future tooling ideas.

## Category-specific rules

### HARD-RULE → CLAUDE.md
- Must be a rule that applies to all future sessions, not project-specific
- Added under the appropriate existing CLAUDE.md section, or proposes a new section if needed
- Must be testable — a future session could observe a violation

### AGENT-IMPROVEMENT → agents/<name>.md
- Must not touch frontmatter
- Must reference evidence from the session where the agent failed or could have been sharper
- Must include the exact existing snippet (not just "change the output section")

### NEW-SKILL → new file
- Proposal includes proposed SKILL.md content as the new text
- Must justify three or more repetitions of the pattern in this session, OR explicit user request
- Must follow Genesis primitives (SKILL.md + optional rules/ + assets/ + commands/)

### PROJECT-FACT → CLAUDE.md (project section)
- Goes under existing "Active Projects & Businesses" section or adds a per-project sub-section
- Must be a durable fact (stack, target audience, known constraint), not a transient state

### ANTI-PATTERN → CLAUDE.md
- Phrased as "do not X" or "avoid X"
- Must reference the moment in this session where doing X caused friction

### GAP-FLAG → no file change, log only
- One-line description of the gap
- Appears only in the learnings doc as a backlog candidate

## Wrapping output

```markdown
# Session Learnings — YYYY-MM-DD

**Session topic:** [One-line summary of what the session was about]
**Duration / scope:** [Rough estimate from session content]
**Proposals generated:** N
**Categories:** [List with counts]

---

[Proposals, one per block, ordered by impact (HARD-RULE and AGENT-IMPROVEMENT first)]

---

## Summary
- Proposals generated: N
- Highest-impact proposal: [name]
- Recommended order of review: [list]
- Gaps flagged (no action this session): [list]
```

## Failure modes to avoid

- **Vague learnings** — "the user prefers good code" is not actionable
- **Single-instance proposals** — one correction is noise; three is signal
- **Voice mismatch** — propose CLAUDE.md text the user wouldn't write themselves
- **Scope drift** — proposing to change agent roles instead of agent behaviours
- **Over-production** — generating 12 proposals from a 15-minute session is suspicious; quality over quantity
